**Bugs found in `arena-lookup-table.c` (from Opus review, `feedback/24-06-2026.md`):**
1. `memcpy(table->entries, new_entry_array, table->capacity)` — wrong arg order (src/dest swapped),
   wrong byte count (should be `* sizeof(entry)`), src gets freed immediately after.
2. `table->capacity` stomped to new size before copy — can't correctly bound the old data copy.
3. Rehash never happens — `populate_lookup_table` only inserts new keys; existing entries keep
   stale indices after a grow. This is the Q1 bug.
4. UB sentinel `0xF364` / `malloc` + read of uninitialized memory. Q3 bug.
5. `arena_lookup_try_get` `while(true)` probe loop has no capacity bound — can loop forever.
6. Dead prototype `get_starting_entry` declared, never defined or used.
